The Mysterious Exploration on the World of UNIX

May 31, 2026 · 5 min read

When I first entered the world of Linux, all I had in mind was that Linux was all about commands. It was never going to be as easy as using Windows. But still, the worst changes to Microsoft Windows, like advertisements across the entire operating system, the forced sync of user files into their cloud, and user privacy concerns, made me take a look at Linux.

When I installed it, I knew nothing about Linux at all. Eventually, I realized that Linux is just a name for the entire core operating system and there are various distributions available for various users. File Systems

A file system is the method and data structure an operating system uses to control how data is stored, organized, named, and retrieved on a storage device like a hard drive or SSD. It acts as the interface between the user and the raw physical storage.

Ext4 (Fourth Extended Filesystem)
The standard, mature journaling file system for most Linux distributions, prioritizing stability and compatibility for general desktop and server use without built-in advanced data management features.
Btrfs (B-Tree Filesystem)
A modern Copy-on-Write (CoW) file system featuring native snapshots, data checksumming for integrity, transparent compression, and built-in RAID, designed for flexible storage management and data protection.
XFS
A high-performance journaling file system optimized for parallel I/O and massive file sizes, commonly used in enterprise environments where raw throughput and scalability are critical.
ZFS (Zettabyte File System)
An enterprise-grade Copy-on-Write file system renowned for maximum data integrity, advanced self-healing capabilities, and powerful volume management, though it requires significant RAM.
F2FS (Flash-Friendly File System)
A lightweight file system specifically designed by Samsung for NAND flash memory (SSDs and mobile storage), optimizing performance and wear leveling on flash-based devices.

Linux Mint

My first choice Linux distribution was Linux Mint, which is similar to the Windows user interface because it uses a Desktop Environment called Cinnamon. After using it for about 4 weeks, I faced some touchpad issues on my new laptop. This made me realize that Linux Mint is based on an LTS version of Ubuntu, so it was built targeting older laptop hardware.

Fedora Workstation

Then, after some research, I found Fedora Workstation. Fedora was amazing and there are many desktop environments that are officially and unofficially (spins) supported by the Fedora team. I used it for around 2 months. It was almost perfect, but what made me switch to another distribution was Fedora's heavy dependency on Flatpaks.

Flatpaks

Flatpaks are great; they have their own isolated space to run apps. But I don't like them personally because Flatpaks are huge in size and never deeply integrated with the operating system when functioning. Some apps show errors and crash when searching for specific files in the file system.

Distro Hopping

Eventually, this turned into a state of Distro Hopping. I was unable to stay on just one distribution. When staying on one distribution, I thought like, "Oh! That distro was better than this, that distro is more stable than this, so I will get fewer app crashes." If any error popped up, I assumed the issue was that specific distro. Not only with distributions, but I also had mental fatigue trying to select the best desktop environment. By trying many distributions, I found that every distro is just based on Debian, Fedora (Redhat), or Arch.

Debian

One of my most stable choices. Updates arrive within a 3-year gap, so their distro stays stable for a long time. Debian has the popular and huge package manager apt. It uses the GNOME DE by default.

CachyOS

We all know about this one. This distro recently got huge hype for its optimized kernels and packages by default. These optimizations make this distro faster on modern laptops. The distro is mainly focused on gaming. It also supports all mainstream desktop environments. Personally, I think the most optimized and default DE is KDE Plasma. It has pacman support just like Arch, and the AUR helper yay is installed by default.

EndeavourOS

It is another Arch-based distro built to make using Arch easy for newbies. It has many pre-installed tools like yay and a pacman cache manager to make daily driving Arch possible. But still, it is based on the Arch (rolling release) model, and there is a high chance things will break.

Finally: Arch Linux

Last and finally, the ultimate rolling release distribution which gives users full control over what they need on their system. Users can build their environment from the base level. Arch supports pacman and the AUR (yay, paru), which have the largest collection of apps when compared to all available package managers.

Package Managers

A package manager helps us install, upgrade, configure, and remove software programs in a Linux distribution. Each distro has its own package manager, and these are not cross-platform supported unless they are based on the same base distributions.

Some popular package managers are:

APT

APT (Advanced Package Tool) is the standard package manager for Debian-based systems (Ubuntu, Mint), managing .deb packages with a focus on stability, user-friendliness, and massive repository support.

DNF

DNF (Dandified Yum) is the modern default for Red Hat-based systems (Fedora, RHEL 8+), managing .rpm packages with improved speed, memory efficiency, and robust dependency resolution compared to its predecessor YUM.

Pacman

Pacman is the lightweight, high-speed package manager for Arch Linux, utilizing simple .pkg.tar.zst archives to facilitate a rolling release model where a single command syncs and upgrades the entire system.

AUR

The AUR (Arch User Repository) is a community-driven collection of package descriptions (PKGBUILDs) that allows users to compile and install software not included in the official Arch Linux repositories.

Yay

Yay (Yet Another Yogurt) is a popular, stable AUR helper written in Go that mimics pacman syntax to automate the downloading, building, and installation of AUR packages with minimal configuration.

Paru

Paru is a modern, high-performance AUR helper written in Rust that enforces mandatory PKGBUILD reviews by default for enhanced security while maintaining full pacman compatibility.

Step-by-Step Installation Guide

  1. Download and Flash ISO

    • Download the ISO file from the official site.
    • Download Balena Etcher and flash the ISO into a USB Drive (4GB+).
    • Restart your computer and press the shortcut to boot into the boot menu (e.g., F12).
    • Choose Arch and boot into it.

    You will be presented with a terminal screen. First, you have to connect to the internet. I am going to show you how to connect to a wireless network like Wi-Fi.

  2. Connect to Internet

    • iwctl
    • device list
    • station *device-name* get-networks
    • station *device-name* connect *network-name*
    • Type the Wi-Fi password.
    • exit

  3. archinstall

    • Mirror: choose your country
    • Disk config: btrfs -> yes -> compression
    • Swap -> no (yes if you need it)

      Note: Swap is an area on your storage drive that your computer uses as overflow memory when your main RAM gets full.

    • Bootloader: Grub

      Note: Grub is stable and mature.

    • Unified Kernel Image (UKI): no

      Note: Complex for newbies.

    • Removable Location: no
    • Authentication: Set your root password and add a user.
    • Profile: KDE Plasma

      Note: Choose what you prefer.

    • Applications:
      • Bluetooth: yes
      • Audio: pipewire
      • Print: no (if you use a printer, then 'yes')
      • Power management: power-profiles-daemon
      • Firewall: ufw
      • Font: Choose noto and noto-emoji fonts by pressing 'space'
    • Network config: default backend
    • Additional applications: choose any apps if you want them preinstalled.

      Note: It's recommended to install plasma-nm (Wi-Fi functionality) and konsole (terminal).

    • Timezone: your timezone

    Save it if you want to use it in the future, then click on install -> yes/ok

Yeah! That's it. Now you can truly enjoy Arch Linux.
